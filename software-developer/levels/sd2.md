---
code: SD2
position: Software Developer
tier: sd
theme: Competence
description: Deepening skills and delivering independently
order: 1
competencies:
  technology:
    keyword: Specializes
    summary: Deepens knowledge in specific functions/languages.
    bullets:
      - Takes initiative to learn new languages, frameworks, methodologies
      - Writes functional and clean code with guidance; consistently follows standards and best practices
      - Has a high level of the technology stack of the project (frontend, backend, devops, etc.) and how it works together
      - Begins to identify common failure modes (edge cases) in their own work
      - Demonstrates high autonomy and minimal error rates on standard, well-defined tasks. While day-to-day velocity increases with limited oversight, the engineer proactively seeks feedback when venturing into complex, high-impact, or ambiguous territory where the risk of systemic error is higher.
  solution:
    keyword: Designs
    summary: Implements solid components from scratch (e.g., RESTful APIs)
    bullets:
      - Successfully delivers the expected functionality for the standard user flow
      - Reduces "Instructional Overhead" - can take a technical requirement and implement it without step-by-step hand-holding
      - Writes functional and clean code with limited errors and increasing velocity
      - Consistently follows standards and requires limited feedback from more senior developers because they have internalized the team’s best practices
      - Displays initiative, highly resourceful
  people:
    keyword: Supports
    summary: Unblocks peers when asked.
    bullets:
      - Provides helpful and professional feedback in code reviews, focusing on team standards
      - Communicates status and roadblocks clearly to the team and stakeholders without needing to be prompted
      - Participates in knowledge sharing, such as helping to document a process or presenting a small topic in a team meeting
      - Collaborates effectively with other disciplines (QA, Design, Product) to ensure feature alignment
  process:
    keyword: Operator
    summary: Navigates and maintains established workflows independently.
    bullets:
      - Consistently follows and enforces the team’s development, testing, and deployment processes
      - Maintains team standards in code reviews, identifying deviations from syntax, library usage, or documentation norms
      - Operates with increased velocity by navigating the project’s internal workflows (CI/CD, environment branching) independently
      - Notices when a specific workflow step is causing recurring delays and brings it to the team’s attention
      - Ensures their work is always accurately represented in tracking tools, providing a clear audit trail for stakeholders
  influence:
    keyword: Collaborator
    summary: Proactively contributes to team success by unblocking peers and providing technical insights that improve the group's collective output.
    bullets:
      - Impacts the team by consistently delivering reliable work that helps the group reach sprint goals
      - Contributes to team ceremonies by offering technical insights and updates that provide clarity to peers and the Lead
      - Provides meaningful peer feedback in code reviews, influencing the quality and standards of the team’s codebase
      - Actively shares knowledge by documenting common fixes or presenting a "how-to" on a specific tool or library the team uses
      - Collaborates across disciplines (e.g., with QA or Design) to ensure that technical implementations align with project requirements
  client:
    keyword: Predictability
    summary: Consistent quality with few regressions.
    bullets:
      - Delivers code that not only works but is robust enough to not cause regressions in existing features
      - Provides a consistent pace of delivery that allows Project Managers to give clients accurate timelines
      - Understands how their changes impact the broader product, ensuring that a fix in one area doesn’t create a fire in another
      - Communicates as soon as they realize a task is more complex than anticipated, preventing "last-minute" surprises for the client
      - Ensures the standard user journey is bulletproof, providing immediate and obvious value to the client’s end users
---
