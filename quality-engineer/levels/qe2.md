---
code: QE2
position: QE Specialist
tier: qe
theme: Evangelizes
description: Researching and introducing new technologies and concepts
order: 1
competencies:
  technology:
    keyword: Evangelizes
    summary: Researches, introduces new technologies and concepts.
    bullets:
      - Demonstrates broad technical scope by conducting observability checks and root cause analysis
      - Regularly fields questions from other testers that recognize them as deeply experienced in a given technology
      - Asks meaningful questions about assumptions or inconsistencies during planning and grooming phases
      - Demonstrates authoritative ownership of projects, consistently delivering results that build stakeholder trust and generate ongoing demand for their leadership
      - Demonstrates knowledge and experience in the various testing types and knowing when to do which
      - Understands the benefits of deeper technical understanding of their work and begins to mentor others
      - Jumps into discussions to provide clarity and support when warranted
      - Displays growing confidence with source control -- will critically review and question what testing must be conducted and what is wasteful as code changes propagate further along toward production
  solution:
    keyword: Designs
    summary: Teaches best practices for PRs and coding practices.
    bullets:
      - Solves issues in others work
      - Navigates complex issues through comprehensive research
      - Demonstrates strong mentorship during code reviews by clearly documenting issues and proactively offering hands-on guidance to junior developers
  people:
    keyword: Supports
    summary: Proactively supports other team members and helps them to be successful.
    bullets:
      - Demonstrates proactivity in problem solving
      - Understands how their role fits into the larger project
      - Calls out risks in requirements
      - When someone is struggling with an issue, jumps on a call to try and figure it out together
      - When a problem or issue is discovered, is able to ask important questions and communicate it and work together to resolve
  process:
    keyword: Enforces
    summary: Models best coding practices, proper GIT flow, proper PR flow.
    bullets:
      - When reviewing a PR is able to communicate proper coding practices for the given language and project
      - Teaches/reinforces others on proper git flow
  influence:
    keyword: Team
    summary: Finds deeper rooted bugs and is able to dispatch them with less help, while communicating to the team what caused it.
    bullets:
      - A bug is found and with testing is able to reproduce with accuracy, and determine the likely reason of the issue
      - Talking with others and working towards a fix efficiently and effectively
  client:
    keyword: Predictability
    summary: Delivers consistent automated coverage integrated into CI/CD pipelines.
    bullets:
      - Builds automated test suites comprehensive enough to catch defects before they reach production
      - Provides predictable quality signals through stable, reliable test automation that Project Managers can trust for release decisions
      - Understands how code changes ripple through the system, ensuring automated regression coverage protects critical paths
      - Communicates proactively when test development will take longer than anticipated, preventing "last-minute" surprises for the client
      - Ensures core user journeys have robust automated coverage, providing immediate confidence to the client's stakeholders
---
